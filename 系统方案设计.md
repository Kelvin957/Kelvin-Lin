# 预约小程序 + 后台管理系统总体方案（Node.js + MySQL）

## 1. 前端开发语言推荐

**推荐：TypeScript（首选）**

原因：
1. 你的业务规则非常复杂（会员、卡券、组合支付、预约规则、设备联动），TypeScript 的类型系统可以显著降低线上逻辑错误。
2. 同时覆盖小程序端和管理端，TypeScript 生态成熟，便于共享类型定义（订单、卡券、会员、计费规则等）。
3. 与 Node.js 后端天然契合（同为 JavaScript/TypeScript 技术栈），前后端协作效率高。

> 实施建议：
> - 小程序端：使用 **Taro + React + TypeScript**（可编译到微信小程序）。
> - 管理后台：使用 **React + TypeScript + Ant Design Pro**。
> - 手机管理端：优先 H5 响应式（同一套 React + TypeScript 代码），后续再按需要封装小程序/APP。

---

## 2. 总体技术架构

- **客户端层**
  - 微信小程序（用户端）
  - Web 管理后台（店长/店员）
  - 手机管理端（H5）
- **网关层**
  - Nginx（lunxi 服务器）
  - HTTPS + 反向代理 + 静态资源托管
- **应用层（Node.js）**
  - API 服务（NestJS/Express）
  - 预约引擎服务（时间表、按小时、按套餐、按时段、按天）
  - 计费引擎服务（会员价、时段价、阶梯价、封顶价）
  - 支付聚合服务（微信、余额、储值卡、优惠券、积分、团购）
  - 卡券中心（次卡/时长卡/期限卡/储值卡/优惠券）
  - 设备控制服务（门锁、电源、空调）
  - 通知中心（站内弹窗、模板消息、短信可选）
  - 第三方渠道服务（美团/抖音核销与预约）
- **数据层**
  - MySQL（主业务库）
  - Redis（缓存、分布式锁、预约库存控制、会话）
  - 对象存储（图片、动图、DIY素材）
- **任务调度层**
  - BullMQ/Agenda + Redis（定时任务：自动取消订单、自动确认到店、提前开门/开空调、订单结束断电）

## 3. 功能模块拆分（对应你的需求）

1. **装修与内容模块**
   - 首页 DIY（模块拖拽、上传图、跳转、7色主题）
   - 预约页顶部装修（会员充值、卡券购买、流程引导）
   - 弹窗公告（首页/预约页/确认页，支持文字/图文/动图+点击动作）
   - 协议管理（强制同意后继续）

2. **预约核心模块**
   - 预约模式：时间表、滚动小时、套餐、按时段、按天
   - 双路径：先选时长后选位置 / 先选位置后选时长
   - 预约规则：起约时间、最小计费时长、打烊限制、预约天数限制
   - 改签/换位/续单/取消/迟到自动取消/提前离店退款

3. **支付与结算模块**
   - 微信支付
   - 余额、储值卡、卡券、优惠券、积分支付
   - 团购支付与核销（抖音、美团）
   - **组合支付引擎**（按优先级拆单：如卡券抵扣后微信补差价）

4. **会员与营销模块**
   - 会员等级、权益、升级奖励、成长值
   - 卡券玩法（次卡、时长卡、期限卡）
   - 储值卡玩法（买赠、有效期、库存、转赠限制）
   - 优惠券玩法（满减、满折）

5. **计费策略模块**
   - 位置价（分钟/小时）
   - 会员价（按等级）
   - 分时段价、节假日价
   - 阶梯价、场次价、封顶价、活动价

6. **商户运营与移动管理模块**
   - 弹窗通知（预约下单/到店/结束/续费/改签/换位/取消等）
   - 手机端查看门店状态、代客预约
   - 财务与用户数据看板
   - 权限组、子账号、多门店账号
   - 区域 WIFI 分配

7. **硬件与物联模块**
   - 门锁：扫码开门、远程开门、常开/常闭、定时任务
   - 电源：随订单通断电/延时断电/定时通断电
   - 空调：远程温控、语音提醒、续费提醒

## 5. 关键技术难点与实现建议

1. **避免超卖/冲突预约**
   - 下单锁定座位与时段时，使用 Redis 分布式锁 + MySQL 事务。

2. **组合支付一致性**
   - 采用“支付单主表 + 支付拆分子表 + 幂等键 + 回调签名校验”。

3. **复杂计费可配置化**
   - 计费规则抽象为策略链，按优先级执行：活动价 > 节假日价 > 会员价 > 基础价。

4. **设备控制可靠性**
   - 所有开门/通电命令落库并可重试，设备状态上报做心跳监控。

5. **第三方平台对账**
   - 美团/抖音核销流水单独建账，支持日对账任务与异常补偿。

## 6. 建议的分阶段实施（务实落地）

### 阶段1（MVP，8~12周）
- 小程序预约主流程（时间表/按小时二选一先做）
- 微信支付 + 余额支付
- 基础会员体系
- 基础后台（订单、用户、门店、价格）
- 门锁远程开门

### 阶段2（增强，6~8周）
- 组合支付（储值卡/优惠券/积分）
- 卡券玩法（次卡/时长卡/期限卡）
- 预约规则完善（改签/换位/续单/取消）
- 财务报表与移动管理端

### 阶段3（高级，6~10周）
- 美团/抖音核销与预约打通
- 全量 DIY 装修与弹窗体系
- 空调、电源联动自动化
- 多门店权限与运营自动化
